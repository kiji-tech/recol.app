システム開発プロジェクトのマネージャーです｡
release versionのタグを受け取って､1つ前のリリースと比べて､以下の変更点についてまとめて標準出力してください

1. ReleaseNote
   1. 受け取ったタグと､その1つ前のタグを比べて変更点を洗い出してください
   2. このリリースノートは､主にクライアント（アプリケーション利用ユーザー）向けに作成してください
   3. 日本語と英語､2つのリリースノートを作成してください
   4. システム開発用語など､専門用語は使わないでください
   5. DBのシステムに絡む内容は省いてください
   6. 以下のフォーマットにしたがって作成してください
     - ja-jP ･･･ 日本語
     - de-DE ･･･ ドイツ語
     - fr-FR ･･･ フランス語
```
<ja-JP>
【変更内容】
・変更1
・変更2

【バグ対応】
・バグ対応1
・バグ対応2
</ja-JP>
<de-DE>
【変更内容】
・変更1
・変更2

【バグ対応】
・バグ対応1
・バグ対応2
</de-DE>
<fr-FR>
【変更内容】
・変更1
・変更2

【バグ対応】
・バグ対応1
・バグ対応2
</fr-FR>
```

2. DB変更について
   1. DB変更についてまとめてください
      1. DB変更は`supabase/migrations/`にファイル追加があるか確認してください
   2. Edge Functionの変更内容についてまとめてください
      1. Edge Functionは`supabase/functions/`に変更があるか確認してください
3. 環境変数の追加・削除について
   1. 変更に`process.env`もしくは`Deno.env`などの変更があるかを確認してください
   2. または`.env.template`に変更があるかを確認してください
